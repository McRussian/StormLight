# Разработка — структура и декомпозиция

## Структура кода

```
StormLight/
├── README.md
├── doc/                        # концепт игры и документация разработки
├── stormlight/                 # основной пакет — игровая модель
│   ├── map/                    # карта: гексы, тайлы, генерация
│   ├── game/                   # механика: колода, ходы, шторм, очки
│   └── simulation/             # симуляция: боты, стратегии, прогон
├── tests/                      # тесты для пакета stormlight
│   ├── map/
│   ├── game/
│   └── simulation/
├── modeling/                   # скрипты для подбора параметров
│   ├── map/                    # эксперименты с генерацией карты
│   ├── game/                   # эксперименты с механикой
│   └── simulation/             # массовые прогоны, сбор метрик
└── prototype/                  # цифровой прототип (будущее)
```

### Роли директорий

**`stormlight/`** — игровой код. Используется и в modeling, и в prototype. Содержит модель игры: карту, механику, логику ботов. Не содержит скриптов запуска и экспериментов.

**`tests/`** — тесты для `stormlight/`. Структура зеркалит подпакеты.

**`modeling/`** — скрипты симуляций. Гоняет партии, собирает метрики в CSV, подбирает параметры. Никакой бизнес-логики — только эксперименты поверх `stormlight/`. Каждый подкаталог содержит README с декомпозицией и чекерами.

**`prototype/`** — цифровой прототип для тестирования механики с живыми игроками. Использует тот же пакет `stormlight/`.
